# 通用问数平台任务列表

## 1. 任务目标

本任务列表基于《通用问数 Agent 平台实施责任分工版》整理，目标是把通用问数平台从方案推进到可开发、可联调、可交付的工程实施阶段。

任务遵循以下原则：

- 先做经营问数标准场景
- 先打通最小闭环
- 先实现平台骨架，再补齐扩展能力
- 核心能力自研，通用能力优先复用

## 2. 任务执行方式

任务按以下方式推进：

1. 先搭工程骨架
2. 再做数据接入
3. 再做语义层和 Agent
4. 再做 SQL 网关和治理
5. 再做评测、监控、运维

## 6. 风险归口

| 风险 | 归口角色 | 处理原则 |
|---|---|---|
| 口径不一致 | 业务口径 Owner | 必须拍板后发布 |
| 数据接入失败 | 数据负责人 | 先保 ODS，再优化同步 |
| 模型命中失败 | 语义/Agent 负责人 | 先追问，再补模型 |
| 权限越界 | 后端负责人 | SQL 网关拦截优先 |
| 接口超时 | 后端负责人 | 降级、限流、缓存 |
| 验收不通过 | 测试负责人 | 形成缺陷单闭环 |

## 7. 一句话总结

通用问数 Agent 平台的实施分工，核心就是让每一层都有明确主责人：产品定范围、数据建底座、语义建模型、后端守安全、前端做体验、测试做评测、运维保稳定，业务口径 Owner 负责把指标说法拍板。
