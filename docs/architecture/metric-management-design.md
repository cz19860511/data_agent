# 指标管理设计

指标管理是通用问数平台的口径中枢，负责把业务上的“销售额、订单数、客单价”等指标定义成可配置、可版本化、可解释的统一对象。

## 1. 目标

- 统一管理平台指标定义
- 支持指标版本化与审核
- 支持指标解释卡片输出
- 支持不同客户的口径差异配置
- 为 Wren、Agent、前端展示提供统一指标来源

## 2. 核心对象

### 2.1 指标定义

指标定义描述“这个指标是什么、怎么算、从哪里来”。

关键字段：
- `metric_code`
- `metric_name`
- `metric_alias`
- `metric_type`
- `base_table`
- `formula`
- `description`
- `owner`

### 2.2 指标版本

指标版本用于记录指标定义的历史演进，避免口径变化造成结果不可追溯。

关键字段：
- `version_id`
- `metric_code`
- `version`
- `change_reason`
- `status`
- `published_at`

### 2.3 指标审核

指标审核用于控制指标发布前的确认流程。

关键字段：
- `audit_id`
- `metric_code`
- `submitter`
- `reviewer`
- `audit_status`
- `audit_comment`

### 2.4 解释卡片

解释卡片是指标面向业务和前端的展示内容，帮助用户理解口径。

关键字段：
- `card_id`
- `metric_code`
- `definition_text`
- `formula_text`
- `scope_text`
- `update_text`

## 3. 指标生命周期

1. 草稿
2. 待审核
3. 已发布
4. 已冻结
5. 已废弃

## 4. 指标管理能力

### 4.1 新建指标

支持定义：
- 指标名称
- 指标编码
- 指标类型
- 指标公式
- 所属主题
- 指标负责人

### 4.2 修改指标

修改时必须产生新版本，不能直接覆盖已发布口径。

### 4.3 审核发布

发布前必须经过：
- 业务口径确认
- 数据负责人确认
- 产品确认

### 4.4 解释输出

每个指标都需要能输出：
- 指标定义
- 计算公式
- 数据来源
- 时间口径
- 适用范围
- 当前版本

## 5. 指标分类

- 核心经营指标
- 过程指标
- 结构指标
- 预警指标
- 辅助说明指标

## 6. 与其他模块的关系

- 与 Doris 主题表对齐指标来源
- 与 Wren 语义层对齐指标名称和可问范围
- 与 SQL 网关对齐指标查询口径
- 与前端展示对齐解释卡内容

## 7. 一期建议

一期优先完成：
- `销售额`
- `订单数`
- `客单价`
- `车流量`
- `毛利率`

先做少量高频核心指标，确保口径稳定后再扩展更多指标。
