# 工程公共规范

这份规范用于约束通用问数 Agent 平台的代码组织、接口风格和日志行为，后续各模块都统一遵循。

## 1. 命名规范

- 服务名使用 `kebab-case`
- Java / 后端包名使用 `lowercase`
- 数据表使用 `snake_case`
- 指标编码使用 `metric_` 前缀
- 维度编码使用 `dim_` 前缀
- 宽表使用 `ads_` 前缀
- 原始接入表使用 `ods_` 前缀

## 2. 接口规范

- 所有 API 统一返回标准结构
- 所有错误必须包含错误码、错误信息、trace id
- 所有请求必须可追踪、可审计
- 所有异步任务必须返回任务 id

## 3. 日志规范

- 关键链路日志必须带 `tenant_id`
- 查询日志必须带 `question_id`
- 执行日志必须带 `request_id`
- 错误日志必须记录可复现上下文

## 4. 配置规范

- 环境变量统一放在 `deploy/env`
- 禁止把敏感信息硬编码到代码中
- 所有外部依赖都必须可配置化

## 5. 数据规范

- ODS 保留原始字段
- DIM 负责统一语义
- ADS 负责面向问数输出
- 指标口径必须落表

## 6. 发布规范

- 所有模型和配置必须版本化
- 发布前必须通过标准问题集回归
- 支持回滚和冻结
